## למה זה קיים

כשמתקינים ריפו — כלי CLI, סקיל, סקריפט — מריצים קוד של מישהו אחר על המחשב שלך. רוב הזמן זה בסדר. לפעמים לא: `curl … | bash` חבוי, מפתח API שדלף, או קובץ סקיל שבשקט אומר לעוזר ה-AI שלך לקרוא את מפתחות ה-SSH שלך.

לקרוא כל שורה בעצמך זה איטי וקל לפספס. **Repo Scanner עושה את הסבב הראשון בשבילך תוך כדקתיים** ונותן דוח ברור לפעול לפיו. הוא אף פעם לא מתקין כלום בלי אישור מפורש ממך.

## מה הוא בודק

| שכבה | מה הוא מחפש |
|------|-------------|
| 🧷 **סקריפטי התקנה** | `curl … \| bash`, `sudo`, שינויי תעודות/רישום, משימות מתוזמנות — כל מה שרץ בהתקנה |
| 🔑 **סודות** | מפתחות API, טוקנים, מפתחות פרטיים — בקבצים הנוכחיים ובהיסטוריית git |
| 📦 **שרשרת אספקה** | חוסר lockfile, תלויות מזויפות, סקריפטי `postinstall` חשודים, **CVE-ים ידועים בתלויות שהוצהרו, דרך OSV.dev** |
| 🧠 **הזרקת פקודות** | הוראות חבויות בקבצי סקיל/קונפיג שמנסות לחטוף סוכן AI — מודל AI מקומי + שכבת חוקים רב-לשונית דטרמיניסטית + רשת סקירה חוצת-שפות אופציונלית |
| 🐍 **קוד פייתון מסוכן** | דפוסי `eval(base64.b64decode(...))` מעורפלים (כולל hop אחד דרך משתנה מקומי), `pickle.loads`/`yaml.load`, `os.system`/`subprocess(shell=True)` עם פקודה דינמית — נתפס ע"י מעבר AST, לא רק regex |
| 🌐 **לאן הקוד מתקשר** | כל כתובת URL — צפויות מול חשודות |
| 👥 **מוניטין** | כוכבים, תורמים, ותק — מכריע-תיקו תחום, לעולם לא "פטור" |

בסוף: ציון 0–100 ודוח מלא לדיסק. כל ממצא מסומן ב**קטגוריית OWASP** שלו (`LLM01 הזרקת פקודות`, `LLM03 שרשרת אספקה`, `LLM06 סוכנות מוגזמת`, `MAL קוד זדוני/RCE` וכו'), כך שסוקר שמכיר את הטרמינולוגיה יכול לקרוא את הדוח במבט אחד. כשה-Claude Code שלך תומך ב-`mcp__visualize__show_widget`, הדוח גם מוצג כקלף ויזואלי קומפקטי בצ'אט לצד ה-Markdown על הדיסק.

## התקנה (צעד אחר צעד)

### שלב 0 — דרישות מוקדמות
- **Git** — לשכפול ריפו. ([git-scm.com](https://git-scm.com/downloads))
- **Python 3.10 ומעלה** — לסורק ה-AI המקומי. ה-installer בודק ואומר לך את הפקודה המדויקת אם חסר (`winget install Python.Python.3.12` ב-Windows, `brew install python` ב-macOS, `apt install python3 python3-venv` ב-Debian/Ubuntu).
- **~1 GB מקום פנוי** — למודל + תלויות, יורד פעם אחת.
- **Claude Code** — זה סקיל של Claude Code; מריצים אותו מתוך Claude Code.

> אתה לא מתקין Python ידנית מעבר לזה שהוא קיים — ה-installer בונה את הסביבה המבודדת, מתקין תלויות, ומוריד את המודל בשבילך.

### שלב 1 — הורד את הקוד
**macOS / Linux**
```bash
git clone https://github.com/Ish-AI1/Repo-Scanner.git
cd Repo-Scanner
```
**Windows (PowerShell)**
```powershell
git clone https://github.com/Ish-AI1/Repo-Scanner.git
cd Repo-Scanner
```

### שלב 2 — (אופציונלי) ודא שה-installer לא שונה
```bash
# macOS/Linux
sha256sum install.sh
# Windows
Get-FileHash install.ps1 -Algorithm SHA256
```
השווה ל-hashes המפורסמים ב-[שלמות](#שלמות-אימות-ה-installer) למטה.

### שלב 3 — הרץ את ה-installer
**macOS / Linux**
```bash
chmod +x install.sh
./install.sh
```
**Windows (PowerShell)**
```powershell
.\install.ps1
```
מה הוא עושה, חי, מול העיניים שלך:
1. מעתיק את הסקיל אל `~/.claude/skills/`
2. מוצא את ה-Python שלך ויוצר סביבה מבודדת (בוחר מיקום נכון למחשב שלך — בלי נתיבים קשיחים)
3. מתקין תלויות
4. **מבקש את אישורך**, ואז מוריד את מודל ה-AI (~1 GB)
5. רושם את נתיב הסביבה כך שהסקיל ימצא אותו אוטומטית
6. מריץ בדיקה עצמית (אמור להופיע `8 passed, 0 failed`)

> **רוצה רק את הסקיל בלי שכבת ה-AI?** `./install.sh --skill-only` (או `.\install.ps1 -SkillOnly`). עדיין אפשר לבדוק עם `/repo-scanner <url> --quick`.

### שלב 4 — טען מחדש והשתמש
ב-Claude Code:
```
/reload-skills
/repo-scanner https://github.com/owner/repo
```

### (אופציונלי) שלב 5 — הוסף את השכבה הרב-לשונית
מוסיף מודל שני, אגנוסטי-שפה (~120 MB, Apache-2.0, בלי חשבון) שמציף ניסיונות הזרקה *בפרפרזה* או *לא-אנגלית* לסקירה. דלג אם אתה בודק רק ריפו באנגלית.
```bash
# השתמש ב-python שה-installer יצר (מודפס בסוף ההתקנה, וגם ב-
# ~/.claude/skills/repo-scanner/.venv-path)
<venv-python> -m pip install sentence-transformers
<venv-python> -c "from sentence_transformers import SentenceTransformer as S; S('sentence-transformers/paraphrase-multilingual-MiniLM-L12-v2'); print('ready')"
```
הסקיל מזהה אותו אוטומטית. בלעדיו, שכבת החוקים הרב-לשונית הדטרמיניסטית עדיין רצה.

## שלמות (אימות ה-installer)

אלה ה-SHA256 של סקריפטי ההתקנה בריפו (סופי-שורה נעוצים ל-LF דרך `.gitattributes`, כך שה-hash זהה בכל מערכת):

| קובץ | SHA256 |
|------|--------|
| `install.sh` | `16b3577b94f7dda85f028a6483872cc8ef09ec9ff10c9afac76be5a30169771e` |
| `install.ps1` | `77b9928701da42845f8c555ad94262a89dc877ddcdf0bbe2068682d59f48d36c` |

אם ה-hash שחישבת לא תואם — **אל תריץ את הסקריפט** — שכפל מחדש או פתח issue.

## איך הוא מנקד

שני צירים, מופרדים בכוונה:
- **סיכון יכולת** — מה שמנגנון ההתקנה *מסוגל לעשות* למחשב שלך.
- **אמון** — מוניטין: מי מתחזק, כמה זמן, תחת כמה עיניים (נדנוד תחום +5/+10, חסום, ו**מתבטל לגמרי כשיש ראיה לזדון** — פופולריות לא מצילה backdoor: `xz`, `event-stream`).

| ורדיקט | ציון | משמעות |
|--------|------|--------|
| 🟢 ירוק | 70–100 | בטוח להתקנה |
| 🟡 צהוב | 55–69 | התקן בזהירות — קרא את הממצאים |
| 🔴 אדום | 0–54 | אל תתקין |

הציון מחושב על ידי קוד (`scorer.py`), לא מנוחש — אותם ממצאים תמיד נותנים אותו מספר.

## הפרטיות שלך

מודלי ה-AI רצים **כולם על המעבד שלך**. שום דבר על הריפו שאתה סורק לא עוזב את המחשב. המודלים בקוד פתוח, יורדים פעם אחת מ-Hugging Face, ונשמרים במטמון. אפשר לעבור ל-offline מלא אחרי ההורדה הראשונה (`TRANSFORMERS_OFFLINE=1`).

## שאלות נפוצות

**צריך לדעת Python?** לא. צריך ש-Python יהיה *קיים*; ה-installer עושה את השאר.

**כלום לא קורה / "command not recognized" כשמדביקים קישור.** Claude Code לא מתקין מ-URL אוטומטית. מתקינים פעם אחת עם `git clone` + `./install.sh`, ואז מריצים `/repo-scanner <url>` בתוך Claude Code.

**אמר "Done" אבל הסריקה אומרת שסורק ה-ML לא זמין.** כנראה בחרת "לא" בפרומפט הורדת המודל, או הרצת `--skill-only`. הרץ שוב `./install.sh` ואשר את ההורדה, או השתמש ב-`--quick`.

**כמה גדולה ההורדה באמת?** ~1 GB פעם אחת (PyTorch CPU + מודל ההזרקה). נשמר לתמיד. השכבה הרב-לשונית האופציונלית מוסיפה ~120 MB.

**אפשר בלי אינטרנט?** אחרי ההורדה הראשונה — כן, `TRANSFORMERS_OFFLINE=1`. שלב ה-clone עדיין צריך רשת.

**זה משנה לי את המערכת?** רק: מעתיק את הסקיל ל-`~/.claude/skills/`, ויוצר סביבת Python אחת תחת תיקיית המשתמש. בלי admin/root, בלי שינוי PATH. הסרה = מחיקת שתי התיקיות.

**למה ריפו פופולרי עדיין ירד לו ניקוד?** מוניטין הוא בונוס קטן, לא פטור. סיכונים אמיתיים (`curl|bash` לא חתום, sandbox כבוי) גוברים עליו במכוון.

**איך מעדכנים?** `git pull` בריפו, להריץ שוב את ה-installer; `/reload-skills` ב-Claude Code.

**הסריקה שלי פרטית?** כן — הסקה מקומית מלאה על ה-CPU, ראה [פרטיות](#הפרטיות-שלך).

## מגבלות ידועות

ביושר, מה הוא *לא* עושה:
- **ניתוח סטטי בלבד.** קורא קוד, לא מריץ אותו. ריפו תמים בהתקנה וזדוני בזמן ריצה יכול לחמוק.
- **ברירת מחדל סורקת docs + agent-config + hooks.** `--deep` לכל הקבצים. ערפול נחוש (payload מקודד/בינארי) עדיין יכול לחמוק מסריקת טקסט.
- **שכבת ה-ML הרב-לשונית היא רשת-recall, לא פסק דין.** מציפה מועמדים ל-Claude לשפוט; לעולם לא מזיזה את הציון לבדה. כיסוי לא-אנגלי הוא best-effort.
- **shallow clone כברירת מחדל** (`--depth 50`). קומיטים — *וגם סריקת היסטוריית-הסודות* — מעבר לעומק הזה לא מכוסים; סוד שהוכנס והוסר מזמן עלול להתפספס.
- זו **חוות-דעת שנייה**, לא ערובה. להתקנות קריטיות — עדיין קרא את הקוד.
